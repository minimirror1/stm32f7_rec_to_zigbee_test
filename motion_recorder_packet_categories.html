PC→MCU 명령
MCU→PC 응답
MCU→PC 이벤트(자동)
재생 제어
PC→MCU 명령모션 재생 (PLAY)
Media 폴더의 특정 모션 파일 재생 요청. MT_RP(반복재생), MT_CT(반복횟수) 설정값과 연계됨.
포함 데이터 → 모션 파일 인덱스 또는 파일명, 반복 횟수 오버라이드 여부
PC→MCU 명령모션 정지 (STOP)
현재 재생 중인 모션 즉시 정지.
PC→MCU 명령모션 일시정지 / 재개 (PAUSE / RESUME)
재생 중 일시정지 및 재개 토글 or 명시적 구분.
재생 상태 — MCU가 PC에 주기적으로 또는 변화 시 전송
MCU→PC 이벤트모션 재생 상태 (PLAY STATUS)
현재 재생 상태를 PC UI에 표시하기 위한 패킷. 상태 변화 시 또는 주기적으로 전송.
포함 데이터 → 현재 상태(재생중/정지/일시정지/초기화중), 현재 재생 파일 인덱스, 현재 반복 횟수/남은 횟수, 진행률(%)
MCU→PC 이벤트모션 재생 완료 (PLAY DONE)
모션 1회 재생이 완료됐을 때 전송. 반복 재생 중이면 반복 카운트 포함.
포함 데이터 → 완료된 파일 인덱스, 완료된 반복 횟수, 남은 반복 횟수
초기화 상태 — 부팅 시 MCU 초기화 과정 모니터링
MCU→PC 이벤트초기화 진행 상태 (INIT STATUS)
MT_ST의 초기화 단계(센서 찍기, 초기위치 이동 등) 진행 상황. Log/BOOT.TXT에 기록되는 부팅 패킷과 연계됨.
포함 데이터 → 초기화 단계(1~4단계), 현재 처리 중인 모터 ID.SUB_id, 성공/실패 여부
MCU→PC 이벤트센서 감지 이벤트 (SENSOR EVENT)
Log/SENSOR.TXT에 기록되는 센서 감지 패킷. MT_AT의 absolute_data=0인 모터 대상.
포함 데이터 → 감지된 모터 ID.SUB_id, 감지 방향(CW/CCW), 감지 값
SD카드 Setting 폴더의 파일들을 PC에서 원격으로 읽고 쓰는 패킷들입니다.
파일별로 읽기/쓰기 패킷을 분리하거나, 파일 ID를 페이로드에 포함해 하나의 읽기/쓰기 패킷으로 통합하는 두 가지 방식이 가능합니다.
단순값 설정 파일 (읽기 + 쓰기)
PC→MCU 명령Discovery ID 읽기/쓰기 (DI_ID)
녹화기 자체 ID 조회 및 변경. Setting/DI_ID.TXT (단일 정수값).
읽기 응답 데이터 → DI_ID 값 (정수)
쓰기 데이터 → 변경할 DI_ID 값
PC→MCU 명령반복 재생 ON/OFF 읽기/쓰기 (MT_RP)
초기화 후 자동 반복재생 여부. Setting/MT_RP.TXT (0 또는 1).
값 → 0: 반복 안 함, 1: 반복 실행
PC→MCU 명령반복 횟수 읽기/쓰기 (MT_CT)
모션 반복재생 빈도. Setting/MT_CT.TXT (정수).
PC→MCU 명령Master/Slave 모드 읽기/쓰기 (MT_MS)
멀티 녹화기 구성 시 역할 설정. Setting/MT_MS.TXT (MASTER / SLAVER 문자열).
PC→MCU 명령리셋 타이머 읽기/쓰기 (RE_TI)
자동 리셋까지의 시간(초). Setting/RE_TI.TXT. 0이면 리셋 안 함.
모터별 설정 파일 (ID.SUB_id 지정 필요)
PC→MCU 명령모터 설정 읽기/쓰기 (MT_ST)
모터 타입(RC/AC/BLDC)별 초기화 파라미터 전체. Setting/MT_ST.TXT.
타입별 구조가 다르므로 타입 구분 필요.
RC: 타입, ID.SUB_id
AC: 타입, ID.SUB_id, 센서방향, 각도, 초기위치, 감속기, 순위
BLDC: 타입, ID.SUB_id, 센서방향, 각도, 초기위치, 감속기, 한바퀴카운트, RPM, 순위
PC→MCU 명령소프트 리밋 읽기/쓰기 (MT_LI)
모터별 소프트웨어 동작 범위 제한. Setting/MT_LI.TXT.
구조: ID.SUB_id, MIN(0~4095), MAX(0~4095)
PC→MCU 명령앱솔루트 모드 읽기/쓰기 (MT_AT)
앱솔루트 동작 모터 목록과 absolute_data. Setting/MT_AT.TXT.
구조: ID.SUB_id, absolute_data (0이면 센서 찍기 수행)
PC→MCU 명령모터 링크 읽기/쓰기 (MT_LK)
두 모터축을 소프트웨어적으로 결합. Setting/MT_LK.TXT.
구조: ID.SUB_id1, offset1, ID.SUB_id2, offset2
Midi 설정 파일
PC→MCU 명령모터 Midi 설정 읽기/쓰기 (Midi/motor)
모터 ID별 별칭(NickName, 8자 이내), 슬라이더 범위(Range), 최소/최대값(LowData/HighData).
파일명 구조: {보드ID}-{모터ID}.ini
포함 데이터 → 보드ID, 모터ID, NickName(8자), Range(0~4095), LowData(0~4095), HighData(0~4095)
PC→MCU 명령Midi 페이지 읽기/쓰기 (Midi/page)
페이지별 CAN ID 목록(최대 8개/페이지), LastPage 정보. Midi/page/data.ini.
포함 데이터 → 페이지 수(cnt), 각 페이지의 ID 배열, LastPage 매핑(CAN ID → 최근 페이지)
연결 및 장치 확인
PC→MCU 명령장치 발견 / Ping (DISCOVERY)
PC가 Zigbee 네트워크에서 MCU를 찾기 위한 브로드캐스트 또는 단일 요청. DI_ID 기반으로 장치를 식별.
응답 데이터 → DI_ID, 펌웨어 버전, MCU 현재 상태, Master/Slave 역할
MCU→PC 응답장치 정보 응답 (DEVICE INFO)
DISCOVERY 요청에 대한 응답. 처음 연결 시 PC가 MCU 전체 구성을 파악하는 데 사용.
포함 데이터 → DI_ID, FW 버전, 현재 상태, MS 역할, 연결된 모터 수
MCU 제어
PC→MCU 명령MCU 리셋 명령 (RESET)
원격으로 MCU를 소프트 리셋. RE_TI 설정과 별개로 즉시 리셋 또는 예약 리셋 가능.
PC→MCU 명령초기화 재실행 명령 (RE-INIT)
MT_ST의 초기화 시퀀스(센서 찍기 → 초기위치 이동)를 리셋 없이 재실행 요청.
오류 및 로그
MCU→PC 이벤트에러 발생 알림 (ERROR EVENT)
Log 폴더에 기록되는 에러 상태를 실시간으로 PC에 전달. 모터 통신 오류, 센서 실패, SD카드 오류 등.
포함 데이터 → 에러 코드, 에러 발생 모터 ID.SUB_id (해당 시), 에러 메시지
PC→MCU 명령로그 읽기 요청 (LOG READ)
Log/BOOT.TXT, Log/SENSOR.TXT 내용을 PC로 전송 요청.
요청 데이터 → 로그 종류(BOOT / SENSOR), 읽을 라인 수 또는 전체
